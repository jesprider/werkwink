# Voice and Style

The report should read like a sharp colleague explaining where things are over
coffee: plain, confident, honest, and quietly interesting. Not a dashboard, not a
press release.

## The four rules

1. **Plain English.** Short sentences. Active voice. Words a new hire would
   understand. If a phrase needs a glossary, rewrite it.
2. **No water.** Cut filler, hedging, and throat-clearing. Every sentence carries
   information. If deleting a sentence loses nothing, delete it.
3. **Not dry.** Translate mechanics into meaning and add the human "why" from the
   notes. Correct-but-boring is a failure too.
4. **Honest.** No spin, no inflation, no invented precision. Confidence where the
   data supports it; plainly say "unclear" where it doesn't.

## Translate mechanics → meaning

Every position fact must become a statement about the work.

| Don't write | Write instead |
|-------------|---------------|
| "Moved from 50 to 70." | "Past the hard part — the approach is settled and we're building." |
| "Position dropped to 45." | "Hit a snag that sent it back to the drawing board." |
| "Sits at 35, +10 this week." | "Still shaping it, but the big unknowns are starting to fall away." |
| "At 99." | "Basically done — just waiting on the last task to close out." |
| "2 active down forces." | "Two things are in the way: the vendor API and a pending security review." |
| "No movement, 4 satellites." | "Hasn't budged in a week — it's stuck waiting on a decision." |

## Kill these words

Corporate filler and empty intensifiers repel readers. Banned unless quoting:

> leverage, synergy, circle back, deep dive, low-hanging fruit, move the needle,
> at the end of the day, in order to, very/really/quite (as padding), various,
> significant (without a number it's vague), utilize (say "use").

Also avoid hedging stacks ("it seems like it might possibly") — commit or say you
don't know.

## Numbers are seasoning

Reach for a number only when it sharpens the point: "stuck for 8 days", "three of
five projects are now in build". Never open with a number, never list them, never
let a number stand in for a sentence of meaning.

## Before / after (a full beat)

**Before (dry metric dump):**

> Billing migration: position 45 (was 62 last week, -17). 2 active down forces.
> 0 snapshots in last 3 days. Owner: Dana.

**After (a report):**

> **Billing migration** got knocked back this week. It looked like a build-it-out
> job until the payment vendor's API changes landed, which forced a rethink — it's
> now back to sorting out the approach. Nothing's moved in three days while Dana
> waits on the vendor's docs. This is the week's main risk; unblocking the vendor
> question is what gets it moving again.

The "after" is longer *because* something happened. A quiet project gets one line,
not a paragraph.

## Self-check before delivery

- Could a stranger read it and know **direction, state, and what's next**?
- Did I delete every sentence that added nothing?
- Did I turn every "N → M" into meaning?
- Is the bad news as clear as the good news?
- Does it sound like a person, not a dashboard?
